Client module¶

class Client.clientclass(dftrip, vm_df, df_LTE, cone)¶

Bases: object

The class clientclass is used to secure all the information required by each client for doing a trip. In this class, methods were added so that the client object could perform some functionalities.

Parameters

  • dftrip (Pandas Dataframe) – Dataframe with the trips

  • vm_df (Pandas Dataframe) – Dataframe with all the user defined VMs

  • df_LTE (Pandas Dataframe) – Dataframe with all the LTE stations

  • cone (Integer) – Cone aperture

  • dfmigrations (Pandas Dataframe) – Dataframe with the source and suggested destination for migration

  • vm (Pandas Dataframe) – Dataframe with the current virtual machine running

  • lte_st (Pandas Dataframe) – Dataframe with the current LTE station that the client is connected with

  • lte_df (Pandas Dataframe) – Dataframe with all the LTE stations

  • latencies (List) – List with the latency that the user has to the source and to the target

  • distancies (List) – List with the distance that the user has to the source and to the target

  • liststats (List) – List with all the statistics from the migrations that occured for that client on that trip

  • mig_id_inc (Integer) – The number of migrations that occured in that trip for that client

  • triptime (Integer) – The time that the whole trip took

  • tripdistance (Integer) – The distance of the whole trip

  • mig_under (Integer) – Defines if the client is under a migration or not

  • timeout (Integer) – The current time of timeout that the client is serving

  • station_heading (Integer) – The heading the station has relative to the client

  • cone – The cone aperture

  • cone_min (Integer) – The minimum value for the cone aperture

  • cone_max (Integer) – The maximum value for the cone aperture
