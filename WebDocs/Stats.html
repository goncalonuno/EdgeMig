Stats module¶

Stats.creatstats(client, server, coor_index)¶

The creatstats function creates and fills a new object of the class statsclass and appends it to the list of stats that is associated to each client. Each obejct and element of that list of stats corresponds to one migration performed

Parameters

  • client (Object of class clientclass) – The client object that has all the information about the client of that trip

  • server (Object of class serverclass) – The server object that the client is currently hosted on

  • coor_index (Integer) – Index of the trips dataframe

Returns

Returns 1 in case of success

Return type

Integer

Stats.path_stats(dfpath, client, coor_index)¶

The path_stats function creates a dataframe with data from the migrations performed related to the latency and distance between the client and the edge server

Parameters

  • dfpath (List) – List of the clients

  • client (Object of class clientclass) – The client object that has all the information about the client of that trip

  • coor_index (Integer) – Index of the trips dataframe

Returns

Returns the dataframe with the statistics

Return type

Pandas Dataframe

Stats.stats_collect(Mt_real, Mt_est, server, client, user_def)¶

The stats_collect function is responsible for collecting the data on every migration that occurs. This data will be usefull determine the effectiveness of the migration mechanisms used

Parameters

  • Mt_real (Float) – Total time of migration

  • Mt_est (Float) – Time estimate for completing the migration

  • server (Object of class serverclass) – The server object that the client is currently hosted on

  • client (Object of class clientclass) – The client object that has all the information about the client of that trip

  • user_def (Object of the class UserDef) – User definitions

Returns

Returns 1 in case of success

Return type

Integer

Stats.stats_df(clientlist)¶

The stats_df function creates a dataframe with all the data saved throughout the running of the simulation. All the data from the migrations performed by each client on each trip is added to the dataframe.

Parameters

clientlist (List) – List of the clients

Returns

Returns the dataframe with the statistics

Return type

Pandas Dataframe

Stats.stats_df_node_dt(clientlist)¶

The stats_df_node_dt function creates a dataframe with data from the migrations performed realted to the client heading and the station heading relative to the client

Parameters

clientlist (List) – List of the clients

Returns

Returns the dataframe with the statistics

class Stats.statsclass(Mt_real, Mt_est, DT_real, mig_id, trip_id, vm_id, lat_source_init, lat_target_init, id_edge_origin, id_edge_target, dist_source_init, dist_target_init, client_heading, station_heading, cone, transferred_data, triptime, tripdistance)¶

Bases: object

The class statsclass is used to save all the information from the multiple migrations that occured in each trip and client. All this data will then be exported and plotted in the plotting program

Parameters

  • Mt_real (Float) – Real migration time

  • Mt_est (Float) – Estimate of the migration time

  • DT_real (Float) – Real downtime

  • mig_id (Integer) – Migration ID

  • trip_id (String) – Trip ID

  • vm_id (Integer) – Virtual machine ID

  • lat_source_init (Float) – Latency from the client to the source as the migration started

  • lat_target_init (Float) – Latency from the client to the target as the migration started

  • id_edge_origin (String) – The ID of the source edge server

  • id_edge_target (String) – The ID of the target edge server

  • dist_source_init (Float) – Distance from the client to the source as the migration started

  • dist_target_init (Float) – Distance from the client to the source as the migration started

  • client_heading (Float) – Heading of the client

  • station_heading (Float) – Heading of the station relative to the client

  • cone (Float) – Cone aperture

  • transferred_data (Float) – Transfered data in GB

  • triptime (Float) – Total time the trip takes

  • tripdistance (Float) – Total distance of a given trip

  • delta_lat_init (Float) – The latency delta between the source and the target once the migration started

  • delta_lat_real_end (Float) – The latency delta between the source and the target once the migration finished (No estimation done)

  • delta_lat_est_end (Float) – The latency delta between the source and the target once the migration finished (With estimation)

  • lat_source_end (Float) – Latency from the client to the source as the migration finished

  • lat_target_end (Float) – Latency from the client to the target as the migration finished

  • delta_dist_init (Float) – The distance delta between the source and the target once the migration started

  • delta_dist_real_end (Float) – The distance delta between the source and the target once the migration finished (No estimation done)

  • delta_dist_est_end (Float) – The distance delta between the source and the target once the migration finished (With estimation)

  • dist_source_end (Float) – Distance from the client to the source as the migration finished

  • dist_target_end (Float) – Distance from the client to the target as the migration finished
