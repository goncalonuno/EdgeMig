user module¶

class user.UserDef(user_def_dict)¶

Bases: object

The class UserDef is used to secure all the information entered by the user when passing the arguments to run the program. This information will impact the way the simulation will run and directly affect the results.

Parameters

  • user_def_dict (Dictionary) – Dictionary with all the user defined paramenters that were entered

  • dftrips_path (String) – Path for the trips dataset

  • dfstations_path (String) – Path for the dataset with the edge server stations

  • df_LTE_path (String) – Path for the dataset with the LTE stations

  • dfvm_path (String) – Path for the dataset with the VMs

  • cone (Integer) – Cone aperture

  • digest_path (String) – Path for outputting the results

  • trip_min (Integer) – The starting trip ID number

  • trip_max (Integer) – The ending trip ID number

  • vm_mode (Integer) – Defines if the simulation will run with all the user designated VMs or if it runs with a single VM depending on ID entered by the user

  • vm_id (Integer) – The ID of the VM if the simulation is ran in vm_mode=0

  • migtype (String) – The type o migration that it was performed

  • benchmark (String) – The type of benchmark that was used while performing the migration

  • PTR (Integer) – Page transfering rate while the migration happend

  • rem_cluster (Integer) – Defines if the simulation will remove edge servers that too close together

  • dynamic_plot (Integer) – Defines if the simulation will display the dynamic plot

  • mig_cost (Integer) – Defines if the simulation will account the cost of migration

  • timeout (Integer) – Defines if the simulation will add a timeout after each migration is performed as a cooldown

  • timeout_multiplier (Integer) – Defines the timout multipler to be used if the user decides to use a timeout upon each migration

  • lat_dist (Integer) – Defines if the simulation will use as a migration policy the distance or latency between the source and target

user.handle_user_def(usr_def)¶

The function handle_user_def is responsible to parse and check if the arguments entered by the user are valid. It returns a dictionary with all the arguments organized in order to then fill the object user_def with all the required definitions.

Parameters

usr_def (List) – List with all the arguments the user entered in the argv

Returns

user_def_dict, a dictionary with all the arguments the user entered

Return type

Dictionary
